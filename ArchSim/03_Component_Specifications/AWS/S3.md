# Component Specification: Amazon S3 (Simple Storage Service)

This document details the configuration parameters, performance limits, and replication properties for simulated Amazon S3 object storage nodes.

---

## 1. Configuration Fields
* **Bucket Region**: e.g., `us-east-1` or `eu-west-1`.
* **Cross-Region Replication (CRR)**: Target region for automatic replication sync.
* **Storage Class**: Standard (hot), IA (infrequent), or Archive.
* **Multipart Upload Limit**: Size threshold (in MB) for chunked uploads.

---

## 2. Runtime State Variables
* **Total Objects size**: Aggregate bytes stored in the bucket.
* **Requests/Sec (GET/PUT)**: Separate counters for read and write queries.
* **Throttling Events**: Occurrences of request rate limits being exceeded.

---

## 3. Failure Modes
* **Prefix Saturation**: Hitting S3 limits (3,500 PUT or 5,500 GET requests per second per prefix) triggers `HTTP 503 Slow Down` throttles.

---

## 4. Simulation Logic
When a client requests a read/write operation:
1. **Network Transit**: Computes packet travel time based on distance (global client to bucket region).
2. **Prefix Lookup**: Evaluates target prefix limits:
   * If requests exceed 5,500 (GET) or 3,500 (PUT) per second inside a single prefix:
     * Returns `HTTP 503 Slow Down`.
3. **Execution Delay**: Latency includes Seek Time and payload size bandwidth constraints.
4. **Replication Sync**: If CRR is enabled, write operations schedule a background `REPLICATE` event to the target bucket region with network delay.
