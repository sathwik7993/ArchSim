# ArchSim Timeline UI Specification

This document details the interface layout and functional interaction rules for the bottom timeline panel, which is used for scheduling failure injection, scrubbing time, and analyzing system behavior history.

---

## 1. Timeline Layout
The timeline resides in a collapsable panel at the bottom of the main viewport. It provides a visual chronological view of the simulation run.

```
+-----------------------------------------------------------------------------+
| [Play/Pause] [Stop] [<<] [>>]  Time: 00:04:12 / 00:10:00 (Virtual)          |
|-----------------------------------------------------------------------------|
| Chaos Track:       [■ DB Partition]      [■ VM Out of Memory]                |
| Alert Track:                [▲ CPU Alert]            [▲ Cache Miss Spike]   |
| Metric Mini-Chart: ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~ |
| Slider Control:    [=============|========================================] |
+-----------------------------------------------------------------------------+
```

---

## 2. Interactive Tracks

### 2.1. Time Slider / Scrubbing Bar
* The user can drag the timeline slider back and forth to scrub through simulation history.
* **Backward Scrubbing**: Retrieves cached historical simulation states from the backend event database and updates the canvas visual elements to represent the chosen historical moment.
* **Forward Scrubbing (Fast Forward)**: If scrubbing to a future point, the Simulation Engine calculates events at maximum execution speed until it reaches the target timestamp.

### 2.2. Chaos Track (Scheduled Chaos Events)
* Displays rectangles representing scheduled system failures.
* Users can add new failures by clicking the timeline and selecting a chaos type.
* Drag-and-drop actions are supported:
  * Dragging blocks changes their start time.
  * Stretching blocks changes failure duration.

### 2.3. Alert Track
* Automatically marks incident events generated by the Metrics and Simulation Engines (e.g., circuit breaker triggers, connection drop, autoscaling triggers).
* Clicking an alert mark moves the timeline slider to that exact timestamp and populates the inspector with details of the component that triggered the alert.

### 2.4. Metric Mini-Chart
* A background micro-sparkline graph running behind the slider bar.
* Displays global indicators like **Aggregate QPS** or **P99 Client Latency**, enabling users to quickly locate latency spikes or traffic drops.
